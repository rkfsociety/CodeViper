## 📋 В планах

Задачи отсортированы от самых лёгких в реализации к самым сложным и пронумерованы
сквозной нумерацией. Уровни сложности: 🟢 мелкая правка → 🟡 надёжность/несколько
файлов → 🟠 новая подсистема внутри приложения → 🔴 сборка бинарников / внешние
сервисы → ⚫ распределённая инфраструктура.

### ⚫ Уровень 5 — распределённая сеть узлов (хостинг, P2P, аккаунты)

Цель: пользователи объединяют мощности ПК — каждый отдаёт до 20% GPU/CPU в общий
пул и пользуется мощностями остальных по принципу «чем больше отдаёшь — тем больше
получаешь».

- [ ] **98. Сигнальный сервер** — VPS с реестром активных узлов (ID, GPU/RAM, нагрузка, онлайн); маршрутизирует соединения между клиентами; Node.js + WebSocket + Redis
- [ ] **99. Аккаунтная система** — регистрация по email/GitHub OAuth; токен при установке определяет репутацию и лимиты
- [ ] **100. Система репутации (баланс мощности)** — узел копит «кредиты» за отданные вычисления, тратит при использовании чужих; нулевой баланс — только свой ПК
- [ ] **101. Режим «Поделиться мощностью»** — переключатель; узел регистрируется на сервере и принимает входящие задачи
- [ ] **102. Лимит ресурсов** — монитор CPU/GPU (`systeminformation`); не более 20% GPU и 15% CPU свободной мощности; при превышении — пауза
- [ ] **103. Очередь входящих задач** — не более 3 одновременно; приоритет узлам с высоким балансом; таймаут 60 сек
- [ ] **104. Маршрутизация исходящих запросов** — сервер ищет свободный узел с нужной моделью и достаточным балансом; иначе локальная Ollama
- [ ] **105. Шифрование трафика** — промпты и ответы между узлами (TLS + end-to-end); владелец узла не видит содержимого чужих запросов
- [ ] **106. Согласие на участие** — явный экран при первом включении: чужие запросы обрабатываются на твоём ПК (зашифровано)
