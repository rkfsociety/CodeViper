### P2P-сеть общих вычислений

Цель: пользователи объединяют мощности ПК — каждый отдаёт до 20% GPU/CPU в общий
пул и пользуется мощностями остальных по принципу «чем больше отдаёшь — тем больше
получаешь».

- [ ] **25. Сигнальный сервер** — VPS с реестром узлов (ID, GPU/RAM, нагрузка); Node.js + WebSocket + Redis
- [ ] **26. Аккаунтная система** — регистрация по email/GitHub OAuth; токен определяет репутацию и лимиты
- [ ] **27. Система репутации** — узел копит кредиты за отданные вычисления, тратит при использовании чужих
- [ ] **28. Режим «Поделиться мощностью»** — переключатель; узел регистрируется и принимает входящие задачи
- [ ] **29. Лимит ресурсов** — `systeminformation`; не более 20% GPU и 15% CPU; при превышении — пауза
- [ ] **30. Очередь входящих задач** — не более 3 одновременно; приоритет по балансу; таймаут 60 с
- [ ] **31. Маршрутизация запросов** — сервер ищет свободный узел с нужной моделью; иначе — локальная Ollama
- [ ] **32. Шифрование трафика** — TLS + end-to-end между узлами; владелец узла не видит чужих промптов
- [ ] **33. Согласие на участие** — явный экран при первом включении
