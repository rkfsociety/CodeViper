### Продукт и UX

### Установщик и саморедактирование через PR

Цель: установленное приложение умеет само себя улучшать — агент правит исходники,
открывает PR, ты его проверяешь и мержишь, приложение автоматически обновляется.

**Архитектура:** установщик кладёт бинарник (`CodeViper.exe`) и рядом клонирует
репозиторий в `source/`. Агент работает с `source/app/...` так же, как сейчас с
`F:\github\CodeViper\app\...`. После пересборки старый `.exe` заменяется новым.

- [ ] **NSIS-установщик** — `electron-builder` собирает `.exe`-установщик; при установке рядом с бинарником клонируется репозиторий (`git clone`) в подпапку `source/`; путь к исходникам сохраняется в настройках приложения
- [ ] **Node.js в комплекте** — установщик включает портативный Node.js (или проверяет системный); без него пересборка после правок невозможна
- [ ] **`gh` CLI в комплекте** — установщик включает портативный `gh` (GitHub CLI); агент использует его для создания веток и PR
- [ ] **Ветки для саморедактирования** — агент создаёт отдельную ветку `agent/fix-...` вместо прямого коммита в `master`; новые инструменты `create_codeviper_branch` + `push_codeviper_branch`
- [ ] **Создание PR** — после пуша агент вызывает `gh pr create` с описанием изменений; PR уходит на ревью к тебе, **не мержится автоматически**
- [ ] **Branch protection на `master`** — в настройках GitHub: запрет прямого пуша в `master`; обязательный CI (typecheck + tests) + ручной approve перед мержем
- [ ] **Статус PR в UI** — панель в интерфейсе со списком открытых PR агента: CI прошёл / упал, ожидает ревью; кнопка открыть PR на GitHub
- [ ] **Пересборка после мержа** — при запуске приложение проверяет, есть ли новые коммиты в `source/` (через `git fetch`); если да — запускает `npm run build` и предлагает перезапуститься с новым бинарником
- [ ] **CI → Release** — GitHub Actions: при мерже PR в `master` собирает новый `.exe` через `electron-builder --publish always` и публикует в GitHub Releases
- [ ] **electron-updater** — альтернативный путь обновления: `electron-updater` проверяет GitHub Releases при каждом запуске; скачивает новый `.exe` в фоне и предлагает перезапуск (работает параллельно с пересборкой из исходников)
- [ ] **Diff перед правками** — preview изменений в UI или инструмент `preview_edit` с подтверждением
- [ ] **Экспорт / импорт чатов** — JSON или Markdown, бэкап и восстановление
- [ ] **Параллельные чаты** — очередь или фоновый run по `chatId` (сейчас один глобальный `agentRunState`)

### Распределённая сеть узлов

Цель: пользователи по всему интернету объединяют мощности своих ПК — каждый отдаёт
до 20% GPU/CPU в общий пул и может пользоваться мощностями остальных.
Система работает по принципу «чем больше отдаёшь — тем больше получаешь».

**Инфраструктура (требует хостинг):**

- [ ] **Сигнальный сервер** — небольшой сервер (VPS) на хостинге; хранит реестр активных узлов (ID, характеристики GPU/RAM, текущая нагрузка, онлайн/офлайн); не обрабатывает запросы — только маршрутизирует соединения между клиентами; стек: Node.js + WebSocket + Redis
- [ ] **Аккаунтная система** — регистрация по email или GitHub OAuth; каждому пользователю выдаётся токен при установке; токен определяет репутацию и лимиты использования чужих ресурсов
- [ ] **Система репутации (баланс мощности)** — узел накапливает «кредиты» за отданные вычисления; тратит кредиты при использовании чужих узлов; нулевой баланс — только свой ПК; высокий баланс — доступ к мощным GPU в сети

**На каждом клиенте (в CodeViper):**

- [ ] **Режим «Поделиться мощностью»** — переключатель в настройках; при включении узел регистрируется на сигнальном сервере и начинает принимать входящие задачи
- [ ] **Лимит ресурсов** — монитор CPU/GPU (через `systeminformation` npm-пакет); жёсткое ограничение: не более 20% GPU и 15% CPU от текущей свободной мощности; при превышении — задача ставится на паузу
- [ ] **Очередь входящих задач** — не более 3 задач одновременно; приоритет отдаётся узлам с высоким балансом репутации; таймаут 60 сек — если не начал, задача уходит другому узлу
- [ ] **Маршрутизация исходящих запросов** — при запросе к модели CodeViper спрашивает сигнальный сервер: есть ли свободный узел с нужной моделью и достаточным балансом? Если да — отправляет туда; если нет — использует локальную Ollama
- [ ] **Шифрование трафика** — все промпты и ответы между узлами шифруются (TLS + end-to-end); владелец принимающего узла не видит содержимого чужих запросов в открытом виде
- [ ] **Согласие на участие** — при первом включении явный экран с предупреждением: чужие запросы обрабатываются на твоём ПК (зашифровано); пользователь явно принимает условия
