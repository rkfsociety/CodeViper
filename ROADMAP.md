## 📋 В планах

### Топ-10 приоритетов

- [ ] **Константы** — вынести магические числа (12 шагов, 85% контекста, 500 ms debounce) в `shared/constants.ts`
- [ ] **Обработка ошибок** — класс `AgentError` и единый механизм обработки
- [ ] **Кэширование** — кэшировать файловое дерево на 5 секунд
- [ ] **Доступность** — ARIA-атрибуты, полноценное управление с клавиатуры
- [ ] **Интеграционные тесты** — полный цикл агента (не только unit-тесты `shared/`)
- [ ] **JSDoc** — документировать публичные API (`shared/`, IPC, инструменты)

### Архитектура и код

- [ ] **Тесты на ядро агента** — `executeTool` (моки fs/spawn), цикл самоулучшения, nudge при JSON-плане в тексте, связка `toolRequiresConfirm` с реальными сценариями
- [ ] **Защита `parseToolArgs`** — разбор аргументов tool call (`agent.ts`, ~стр. 409) сейчас вне `try/catch` вокруг `executeTool`: битый JSON у слабых Ollama-моделей роняет весь прогон. Обернуть и при провале вернуть модели tool-сообщение для повтора

### Продукт и UX

- [ ] **Установщик** — `electron-builder`, portable `.exe` / NSIS, автообновления (сейчас запуск через `npm run dev`)
- [ ] **Diff перед правками** — preview изменений в UI или инструмент `preview_edit` с подтверждением
- [ ] **Экспорт / импорт чатов** — JSON или Markdown, бэкап и восстановление
- [ ] **Параллельные чаты** — очередь или фоновый run по `chatId` (сейчас один глобальный `agentRunState`)

### Агент и AI

- [ ] **Anti-loop в плане самоулучшения** — счётчик попыток на каждый пункт плана; после 3 неудач подряд пункт переходит в состояние `blocked` с причиной, агент перестаёт крутиться
- [ ] **Детектор опасных задач** — анализ текста задачи до запуска агента; при совпадении с паттернами (удаление, массовые операции) показывает предупреждение и требует подтверждения
- [ ] **Лимит шагов и прогонов** — настройка `maxStepsPerRun` и `maxRunsPerHour`; защита от бесконечного цикла самоулучшения и перегрева Ollama
- [x] **Параллельные tool calls** — независимые `read_file` / `grep_files` выполнять одновременно
- [ ] **Семантический поиск** — эмбеддинги через Ollama (`nomic-embed-text`) + локальный индекс в `.codeviper/`
- [ ] **MCP (Model Context Protocol)** — подключение внешних источников (БД, API, браузер) без своих инструментов под каждый случай
- [x] **Частичное чтение больших файлов** — `read_file` с offset/limit по строкам вместо ошибки на файлах >500 KB (агент сможет работать с крупными файлами)

### Безопасность

- [ ] **Режим по умолчанию** — `acceptEdits` или `ask` вместо `bypass` + предупреждение при первом запуске
- [ ] **Усилить песочницу команд** — whitelist безопасных команд, запрет запуска `.ps1`/`.bat` без подтверждения, `sandbox: true` в `webPreferences`
- [ ] **Денилист команд — лишь страховка** — `BLOCKED_PATTERNS` (`services.ts`) принципиально обходится (`rm -r -f`, `python -c …`); реальная граница — `permissionMode` + подтверждения, не полагаться на блок-лист
- [ ] **Настройка git-sync при старте** — вкл/выкл или предупреждение (сейчас `git stash` + `reset --hard` в `start-dev.ps1`)

### Инфраструктура

- [ ] **E2E-тесты** — smoke через Playwright для Electron (создать чат, ping Ollama)
- [ ] **ESLint + Prettier** — единый стиль и статический анализ (особенно важно из-за саморедактирования исходников агентом)
- [ ] **Hot reload main process** — инструмент `restart_app` или авто-reload `electron/main/*` в dev
- [ ] **Кроссплатформенность** — `.sh` launcher, тесты Linux/macOS в CI (сейчас Windows-first)

### Быстрые победы

- [x] **Индикатор clarifyMode** — явная подсказка «агент ждёт ответа на уточнение»
- [ ] **Настраиваемый таймаут команд** — сейчас фиксированные 120 с
