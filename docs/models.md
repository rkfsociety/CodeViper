# Модели Ollama

## Рекомендуемые модели

| RAM | Рекомендуем | Альтернативы |
|---|---|---|
| **6–8 GB** | `qwen2.5-coder:3b` ★ | `llama3.2:3b`, `qwen3:4b` |
| **8 GB** | `qwen2.5-coder:7b` ★ | `qwen3:8b`, `llama3.1:8b`, `qwen2.5:7b` |
| **12 GB** | `mistral-nemo:12b` | — |
| **16 GB** | `qwen2.5-coder:14b` ★ | `qwen3:14b`, `qwen2.5:14b`, `codestral:22b` |
| **24–32 GB** | `qwen2.5-coder:32b` ★ | `qwen3:30b` |
| **48 GB+** | `llama3.3:70b` ★ | `llama3.1:70b`, `qwen2.5:72b` |

★ — лучший выбор в категории.

Модели **без tool calling** не подходят для агента: `llama2`, `gemma2`, `deepseek-r1`, `mistral:7b`. В каталоге они не показываются; уже установленным — предложение удалить.

Скачать модели можно прямо из CodeViper: **Настройки → каталог моделей**. Уже установленные не дублируются. Несколько моделей качаются по очереди («В очередь»), прогресс виден в верхней панели.

## Автовыбор модели (режим Auto)

По умолчанию включён **автовыбор**. При нескольких установленных моделях перед каждым запросом агент:

1. Оценивает сложность задачи (короткий вопрос → лёгкая, рефакторинг → тяжёлая)
2. Выбирает лучшую установленную модель с учётом минимального размера: задачи сложности ≥60 — не ниже 7B, ≥75 — не ниже 14B
3. Выгружает другие модели из RAM Ollama (`keep_alive: 0`)
4. Пишет в чат: «🤖 Авто-модель: …» с причиной выбора

Отключить: **Настройки → снять «Автовыбор модели»** — тогда всегда используется модель из списка.

## Эскалация при рефьюзале

Если модель отвечает «я не могу» / «как AI-агент» вместо использования инструментов — агент автоматически переключается на следующую по размеру установленную модель и повторяет шаг (один раз за сессию).
