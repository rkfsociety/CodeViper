# Использование

## Первые шаги

1. Нажми **«+ Чат»** слева — создаётся новый чат
2. Нажми **«Выбрать проект»** и укажи папку с кодом (или создай папку со привязанным проектом — см. ниже)
3. Убедись, что индикатор Ollama **зелёный**
4. Опиши задачу — агент сам прочитает файлы, внесёт правки, запустит команды

> **Два разных «проекта»:**
> - **CodeViper** — папка `app/` (исходники приложения)
> - **Рабочий проект** — папка, которую выбираешь в чате; у каждого чата свой проект; после первого сообщения сменить нельзя

## Папки чатов и проекты

Папка чатов может иметь **привязанный проект**: кнопка 📂 в заголовке папки открывает диалог выбора, путь сохраняется в папке. Все новые чаты в папке автоматически получают этот проект — выбирать вручную не нужно.

## История чатов

- **Папки** для группировки; поиск по названию и проекту
- **Drag-and-drop** — перетащи чат на папку или в «Без папки»
- Переименование и удаление через модальные окна
- Автосохранение с debounce 500 ms + flush при смене чата
- Данные: `%APPDATA%/CodeViper/chats/index.json` и `chats/data/{id}.json`

## Очередь и управление агентом

- Пока модель отвечает — можно продолжать писать, сообщения попадают в **очередь**
- Кнопка **«Стоп»** — прерывает текущий запрос и очищает очередь
- **Один активный run** — параллельные запросы блокируются
- Статус-бар: фаза, tok/s, время шага; при суммаризации — «Σ сжимаю…»

## Контекст модели

Панель **«Контекст»** над чатом показывает % лимита и секции. При ≥85% — предупреждение. Агент автоматически **суммаризирует** историю через Ollama (самая лёгкая установленная модель). Клик открывает детальный просмотр секций.

## Настройки

- **Настройки** (кнопка в верхней панели): модель, Ollama URL, суммаризация, самообучение, навыки, лимит шагов
- Настройки сохраняются в `%APPDATA%/CodeViper/settings.json`
- Позиция и размер окна — `%APPDATA%/CodeViper/window-state.json`

## Частые вопросы

**Модель советует, но не делает сама:**

| Причина | Что делать |
|---|---|
| Модель без tool calling | Выбрать `qwen2.5-coder:7b` или `llama3.1:8b` |
| Слабая модель + сложная задача | Поставить модель ≥7B, автовыбор переключится |
| Задача про Figma, Material-UI и т.п. | Переформулировать: «**Прочитай** `styles.css` и **измени** …» |
| Правка самого CodeViper при открытом другом проекте | Явно указать: «используй `write_codeviper_file`» |

При повторе в чате появится: *«Модель ответила текстом без инструментов — повторяю…»*
