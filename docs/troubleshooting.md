# Устранение неполадок

Короткие решения для самых частых проблем CodeViper. Если приложение не стартует как ожидается, сначала попробуйте запуск через `CodeViper.cmd console` и посмотрите текст ошибки в консоли.

## Чёрный экран или пустое окно

Если окно запускается, но интерфейс остаётся пустым или чёрным:

- Полностью закройте CodeViper и запустите его снова.
- Попробуйте `CodeViper.cmd console`, чтобы увидеть ошибки запуска.
- Проверьте, не блокирует ли система запись GPU-кэша.

В приложении уже отключён дисковый GPU shader cache для Windows-систем, где `GPUCache` иногда падает с `Access denied` и даёт чёрный экран. Если проблема повторяется, обычно помогает:

- удалить кэш приложения в `%APPDATA%\CodeViper\GPUCache`, если папка осталась после сбоя;
- убедиться, что антивирус или защитник Windows не блокирует `%APPDATA%\CodeViper`;
- перезапустить приложение после очистки кэша.

## Проблемы с плагинами

Плагины загружаются из `~/.codeviper/plugins`.

Проверьте следующее:

- в папке лежат файлы `.js`, а не `.ts`;
- экспорт плагина имеет корректную структуру: имя, описание и список tools;
- после добавления или правки плагина приложение перезапущено.

Если плагин не подхватывается:

- откройте папку через Настройки → Плагины;
- проверьте лог в консоли запуска;
- убедитесь, что файл не содержит синтаксических ошибок и действительно компилирован в `.js`.

## Portable Node

CodeViper использует portable Node.js для live runtime и сборки packaged-версии. Он скачивается командой `npm run setup-node` в `app/resources/node/` и не должен коммититься в репозиторий.

Если runtime не может собрать `app/`:

- проверьте, что `app/resources/node/` существует после `npm run setup-node`;
- для packaged-версии проверьте, что portable Node попадает в `resources/node/`;
- если папка повреждена, удалите `app/resources/node/` и заново выполните `npm run setup-node`.

## Когда помогает `CodeViper.cmd console`

Запуск через `CodeViper.cmd console` полезен, если:

- окно не появляется;
- после обновления runtime что-то ломается на старте;
- плагин или live runtime падает без понятного сообщения в UI.

Этот режим быстрее всего показывает, это проблема Electron, runtime-сборки, плагина или локального окружения.
