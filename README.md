# 🐍 CodeViper

**CodeViper** — локальный AI-агент с GUI для программирования. Работает полностью на вашем компьютере через [Ollama](https://ollama.com). Никаких подписок, API-ключей и облачных лимитов.

![stack](https://img.shields.io/badge/Electron-36-47848F) ![stack](https://img.shields.io/badge/React-19-61DAFB) ![stack](https://img.shields.io/badge/Ollama-local-7EE787)

## Возможности

- 💬 **Чат-агент** — ставит задачи, планирует, отвечает на русском; ответы приходят **потоком** в реальном времени
- 📚 **История чатов** — папки, сохранение переписки, переключение между задачами
- 📁 **Файлы** — агент читает и редактирует файлы внутри открытого проекта
- 🖥️ **Терминал** — запускает команды в корне проекта
- 🔒 **Приватность** — код не уходит в облако
- 🧠 **Модели Ollama** — выбор установленных и скачивание рекомендуемых прямо из приложения
- 🧬 **Самообучение** — агент запоминает уроки, предпочтения и правила проекта между сессиями

## Быстрый старт

### 1. Установите Ollama

Скачайте и установите [Ollama](https://ollama.com). После установки приложение Ollama должно быть запущено (иконка в трее).

Модели можно скачать **прямо в CodeViper** — в панели «Настройки» справа есть список рекомендуемых моделей и кнопка «Скачать».

Рекомендуемые модели с поддержкой tool calling:
- `qwen2.5-coder:7b` — быстро, 8 GB RAM
- `qwen2.5-coder:14b` — лучше качество, 16 GB RAM
- `llama3.1:8b` — универсальная с tool calling

### 2. Запуск через .exe (рекомендуется)

**Пересборка exe** — скрипт в корне проекта или кнопка **«Пересобрать exe»** в настройках (при запуске через `npm run dev`):

```powershell
# Двойной клик или из терминала:
.\rebuild-exe.bat

# Альтернатива через npm:
npm run rebuild:exe
```

Сборка создаёт в корне:
- `CodeViper.exe` — portable, без установки (просто запусти)
- `CodeViper-Setup.exe` — установщик NSIS

**Важно:** Ollama должна быть установлена и запущена отдельно — CodeViper подключается к ней по `http://127.0.0.1:11434`.

### 3. Запуск из исходников (для разработки)

```powershell
npm run dev
```

### 4. Использование

1. Нажмите **«Открыть проект»** и выберите папку с кодом
2. Слева создайте **новый чат** или выберите из истории (можно группировать в **папки**)
3. Убедитесь, что индикатор Ollama **зелёный**
4. Скачайте модель (если ещё нет) или выберите установленную в настройках справа
5. Опишите задачу в чате — агент сам прочитает файлы, внесёт правки и запустит команды

## История чатов

- Слева — список чатов и **папки** для группировки
- Каждый чат привязан к проекту и **сохраняется автоматически**
- Хранилище: `%APPDATA%/CodeViper/chats.json`
- Фильтр «Все чаты» / только текущий проект
- Переименование, удаление, перенос чата между папками

## Самообучение

Агент накапливает опыт локально и использует его в следующих задачах:

| Что | Где хранится |
|---|---|
| Глобальные знания (паттерны, навыки) | `%APPDATA%/CodeViper/memory.json` |
| Знания проекта | `{проект}/.codeviper/memory.json` |
| Правила проекта | `{проект}/.codeviper/rules.md` |

**Как это работает:**
1. Перед задачей агент подгружает релевантные знания в системный промпт
2. Во время работы может вызвать `remember`, `search_memory`, `forget`
3. После задачи с инструментами — автоматическая рефлексия (если включено «Самообучение»)
4. Можно сказать: «запомни, что мы используем tabs, не spaces» — агент сохранит это

Память можно просмотреть и удалить в панели «Настройки» → «Память агента».

## Архитектура

```
CodeViper/
├── electron/
│   ├── main/          # Main process: агент, Ollama, файлы, терминал
│   └── preload/       # Безопасный мост IPC
└── src/               # React GUI
    └── components/    # Чат, история, терминал
```

Агент использует цикл **ReAct**:
1. Отправляет запрос в Ollama с инструментами и памятью
2. Модель вызывает `read_file`, `write_file`, `list_directory`, `run_command`, `remember`, `search_memory`
3. Результаты возвращаются модели
4. Повторяется до завершения задачи (макс. 12 шагов по умолчанию)
5. После задачи — рефлексия и сохранение уроков (если самообучение включено)

## Безопасность

- Агент работает **только внутри** выбранной папки проекта
- Файлы больше 500 KB не читаются
- Опасные команды (`rm -rf`, `format`, `shutdown`) заблокированы
- Команды в терминале автоматически прерываются через **120 секунд** (exit code 124)
- Все данные остаются на вашем ПК
