# CodeViper

**CodeViper** — локальный AI-агент с GUI для программирования. Работает полностью на вашем компьютере через [Ollama](https://ollama.com). Никаких подписок, API-ключей и облачных лимитов.

Встроенного редактора кода нет: вы описываете задачу в чате, агент сам читает файлы, правит их и запускает команды в терминале.

## Возможности

- 💬 **Чат-агент** — ставит задачи, планирует, отвечает на русском; ответы приходят **потоком** с **Markdown**-разметкой; кнопка **«Копировать»** у каждого сообщения
- 📚 **История чатов** — папки, сохранение переписки, переключение между задачами
- 📁 **Файлы** — агент читает, **создаёт** (`create_file`), **редактирует точечно** (`edit_file`), перезаписывает (`write_file`) и дописывает (`append_file`) файлы в проекте
- 🖥️ **Терминал** — запускает команды в корне проекта
- 🔒 **Приватность** — код не уходит в облако
- 🧠 **Модели Ollama** — выбор установленных и скачивание рекомендуемых прямо из приложения
- 🧬 **Самообучение** — агент запоминает уроки, предпочтения и правила проекта между сессиями
- 🎨 **Интерфейс** — карточки шаблонов задач, быстрые подсказки, бейджи ролей в чате, статус модели/Ollama в шапке

## Быстрый старт

### 1. Установите Ollama

Скачайте и установите [Ollama](https://ollama.com). После установки приложение Ollama должно быть запущено (иконка в трее).

Модели можно скачать **прямо в CodeViper** — кнопка **«Настройки»** в верхней панели, там список рекомендуемых моделей и кнопка «Скачать».

Рекомендуемые модели с поддержкой tool calling:
- `qwen2.5-coder:7b` — быстро, 8 GB RAM
- `qwen2.5-coder:14b` — лучше качество, 16 GB RAM
- `llama3.1:8b` — универсальная с tool calling

### 2. Запуск

**Один раз** после клонирования (или просто первый двойной клик — скрипт сам сделает `npm install`):

```powershell
cd app
npm install
```

**Повседневный запуск** — двойной клик по **`CodeViper.cmd`** в **корне репозитория** (рядом с папкой `app/`). Консоль не показывается; при ошибке появится окно с подсказкой.

- Отладка с консолью: `CodeViper.cmd console`
- Лог launcher: `%LOCALAPPDATA%\CodeViper\launch.log`
- Лог `npm run dev`: `%LOCALAPPDATA%\CodeViper\dev.log`

Если двойной клик «ничего не делает» — запустите `CodeViper.cmd console` и смотрите вывод в терминале.

Ollama должна быть установлена и запущена отдельно — CodeViper подключается к ней по `http://127.0.0.1:11434`.

### 3. Использование

1. Слева нажми **«+ Чат»** — создай новый чат
2. В чате нажми **«Выбрать проект»** и укажи папку с кодом
3. Убедись, что индикатор Ollama **зелёный**
4. Скачай модель (если ещё нет) или выбери установленную в **Настройках**
5. Опиши задачу — агент сам прочитает файлы, внесёт правки и запустит команды

**Два разных «проекта»:**
- **CodeViper** — папка `app/` (исходники приложения)
- **Рабочий проект** — папка, которую выбираешь **внутри чата**; у каждого чата свой проект. После **первого сообщения** сменить проект нельзя

## История чатов

- Слева — список чатов и **папки** для группировки
- **Поиск** по названию чата и проекту
- **Drag-and-drop** — перетащи чат на папку или в «Без папки»
- Переименование и удаление через **модальные окна** (не `prompt`/`confirm`)
- Каждый чат привязан к **своему проекту** и **сохраняется автоматически** (debounce 500 ms + flush при смене чата)
- Данные чатов: `%APPDATA%/CodeViper/chats/index.json` (метаданные) и `chats/data/{id}.json` (сообщения); старый `chats.json` мигрируется автоматически
- **Стрим агента привязан к chatId** — события попадают только в тот чат, где запущена задача
- **Один активный run-agent** — параллельные запросы блокируются до завершения текущего
- Кнопка **«Стоп»** прерывает текущий запрос агента (отмена fetch к Ollama) и очищает очередь
- Пока модель отвечает, можно **продолжать писать** — новые сообщения попадают в **очередь** и обрабатываются по порядку
- Длинная история **автоматически обрезается** под лимит контекста модели (~28k символов)

## Настройки

- Кнопка **«Настройки»** в верхней панели — модель, Ollama URL, самообучение, навыки
- Настройки **сохраняются между сессиями**: `%APPDATA%/CodeViper/settings.json`
  (модель, URL Ollama, лимит шагов, флаг самообучения)
- Позиция и размер окна запоминаются в `%APPDATA%/CodeViper/window-state.json`
- Память и навыки в настройках показываются для **текущего чата** (его проекта)

## Самообучение

Агент накапливает опыт локально и использует его в следующих задачах:

| Что | Где хранится |
|---|---|
| Глобальные знания (паттерны, навыки) | `%APPDATA%/CodeViper/ViperMemory.md` |
| Знания проекта | `{проект}/.codeviper/ViperMemory.md` |
| Правила проекта | `{проект}/.codeviper/rules.md` |
| **Навыки (skills)** | `%APPDATA%/CodeViper/ViperSkills.md` (global, по умолчанию) и `{проект}/.codeviper/ViperSkills.md` (scope: project) |
| **Viper Memory** (встроенный skill) | `viper-memory` — remember / search_memory / forget → **ViperMemory.md** |
| **Viper Model Training** (skill) | `viper-model-training` — адаптация Ollama через Modelfile + few-shot |
| **Viper Agent Core** | `viper-agent-core` — полный workflow и список инструментов |
| **Viper Files** | `viper-files` — read/create/edit/append/write файлов |
| **Viper Codebase** | `viper-codebase` — list_directory, grep_files, find_files |
| **Viper Terminal** | `viper-terminal` — run_command |
| **Viper Self-Edit** | `viper-self-edit` — правка исходников CodeViper |
| **Viper Self-Improvement** | `viper-self-improvement` — план автономного самоулучшения |
| **Данные навыков** (todo, состояние) | `%APPDATA%/CodeViper/skill-data/` и `{проект}/.codeviper/skill-data/` |

**Как это работает:**
1. Перед задачей агент подгружает релевантные знания и **активные skills** в системный промпт
2. Во время работы: `remember`, `search_memory`, `forget`
3. **По запросу** («сделай skill для todo», «улучши себя»): `create_skill` (по умолчанию **global** → ViperSkills.md), `update_skill`, `read_skill`
4. После задачи с инструментами — автоматическая рефлексия (если включено «Самообучение»)

**Примеры запросов:**
```
Сделай себе skill для todo-листа в этом проекте
```
```
Улучши себя: добавь skill для code review перед коммитом
```

Память и навыки можно просмотреть в **Настройках** (кнопка в верхней панели). Терминал — кнопка **«Терминал»** под чатом.

### Саморедактирование CodeViper

Агент может улучшать **свой** код и навыки (не только проект в чате):

| Действие | Инструмент |
|---|---|
| Навык (инструкции) | `create_skill` с `scope: global` |
| Новый файл в app/ | `create_codeviper_file` |
| Точечная правка | `edit_codeviper_file` (old_string → new_string) |
| Полная перезапись | `write_codeviper_file` |
| Дописать в конец | `append_codeviper_file` |
| Читать исходники | `read_codeviper_file` |
| Тесты после правок | `run_codeviper_command` → `npm run typecheck`, `npm test` |
| Структура приложения | `list_codeviper_directory` |
| Поиск в коде | `grep_codeviper_files`, `find_codeviper_files` |

Примеры:
```
Изучи код и начни улучшать себя
```

Агент войдёт в **режим автономного самоулучшения**:
1. Изучит исходники CodeViper (`list_codeviper_directory`, `read_codeviper_file`)
2. Создаст план через `set_self_improvement_plan` (3–8 пунктов)
3. Будет выполнять пункты **до конца** — после каждого `complete_self_improvement_item`
4. Остановится только когда все пункты done или достигнут лимит шагов (мин. 20)

Прогресс плана отображается системными сообщениями в чате.

```
Улучши себя: добавь skill для code review
```
```
Добавь в agent.ts поддержку …
```

После правок `electron/main/*` нужен **перезапуск** приложения.

### Адаптация моделей Ollama (Viper Model Training)

CodeViper **не выполняет классический GPU fine-tuning**. Вместо этого агент может создать **производную модель Ollama** из файла примеров:

1. Подготовьте JSON/JSONL: `[{"user":"…","assistant":"…"}]` → `.codeviper/training/examples.json`
2. В чате: «обучи модель на examples.json» или «создай модель my-coder из qwen2.5-coder:7b»
3. Агент вызовет `preview_ollama_modelfile`, затем `create_ollama_model`
4. Выберите новую модель в **Настройках**

Инструменты: `preview_ollama_modelfile`, `create_ollama_model`. Встроенный skill: `viper-model-training`.

## Архитектура

```
CodeViper/
├── README.md
├── CodeViper.cmd      # запуск приложения
└── app/               # исходники
    ├── resources/     # иконка приложения
    ├── shared/        # общие утилиты (makeId)
    ├── electron/      # Main process: агент, Ollama, файлы, терминал
    └── src/           # React GUI
```

Агент использует цикл **ReAct**:
1. Отправляет запрос в Ollama с инструментами, памятью, **путём проекта**, **структурой файлов** и **полной историей чата**
2. Модель вызывает `list_directory`, `grep_files`, `find_files`, `read_file`, `create_file`, `edit_file`, `write_file`, `run_command`, …

### Просмотр контекста (как в Cursor)

Над чатом — панель **«Контекст»** с оценкой ~токенов и чипами секций. Клик открывает модальное окно:

| Секция | Что внутри |
|---|---|
| Инструкции агента | Базовый system prompt |
| Саморедактирование | Пути и workflow CodeViper |
| Проект | Корень и дерево файлов |
| ViperMemory и навыки | Релевантные записи и skills |
| Инструменты | Список function tools для Ollama |
| Сообщения | System + история + текущий запрос |

Контекст обновляется при наборе сообщения (черновик) и фиксируется при отправке запроса.

### Почему модель «советует», но не делает сама

CodeViper **не выполняет** ваш запрос отдельно от модели — агент только передаёт текст в Ollama и **запускает инструменты, если модель их вызвала**. Частые причины «плана вместо действия»:

| Причина | Что делать |
|---|---|
| Модель без tool calling (llama2, mistral 7b и т.п.) | В настройках: `qwen2.5-coder:7b` или `llama3.1:8b` |
| Модель отвечает как консультант (Figma, Material-UI, «сделайте сами») | Переформулируйте: «**Прочитай** `styles.css` и **измени** …»; агент повторит запрос с обязательным tool call |
| Правка **самого CodeViper**, а открыт другой проект | Скажите явно: «используй `write_codeviper_file` для UI CodeViper» |
| Обрыв JSON в ответе (`}` в конце) | Слабая модель пыталась написать tool call текстом — смените модель |

В чате при повторе появится: *«Модель ответила текстом без инструментов — повторяю…»*. Если после повтора инструментов нет — смените модель или уточните задачу.

3. Результаты возвращаются модели
4. Повторяется до завершения задачи (макс. 12 шагов по умолчанию)
5. После задачи — рефлексия и сохранение уроков (если самообучение включено)

## Безопасность

- Агент работает **только внутри** выбранной папки проекта
- Файлы больше 500 KB не читаются
- Опасные команды блокируются (rm -rf, format, shutdown, diskpart, encoded powershell и др.)
- `run_command` запускается через `cmd.exe` / `sh` **без** `shell: true`; лимит длины команды — 4096 символов
- Команды в терминале автоматически прерываются через **120 секунд** (exit code 124)
- Все данные остаются на вашем ПК

## Сравнение с Cursor

| | Cursor Pro | CodeViper |
|---|---|---|
| Цена | $20/мес | $0 |
| Лимиты | Месячные пулы | Нет (железо) |
| Качество AI | ⭐⭐⭐⭐⭐ | ⭐⭐⭐ (зависит от модели) |
