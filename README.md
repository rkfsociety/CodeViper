Вы описываете задачу в чате — агент сам читает файлы, правит их и запускает команды в терминале. Всё на вашем компьютере: данные никуда не уходят.

## Требования

| | |
|---|---|
| Windows 10 / 11 | ✅ |
| Node.js | 18+ |
| RAM | 8 ГБ (16 ГБ для локальных моделей) |
| [Ollama](https://ollama.com) | для локальных моделей |
| DeepSeek / OpenAI API key | для облачных моделей |

## Быстрый старт

**Первый раз:**
```powershell
cd app && npm install
```

**Запуск:** двойной клик по **`CodeViper.cmd`** в корне репозитория.  
С консолью: `CodeViper.cmd console`

После запуска:
1. **«+ Чат»** — создать чат, выбрать папку проекта
2. Выбрать провайдер модели в настройках (Ollama / DeepSeek / OpenAI)
3. Описать задачу

## Возможности

### Агент
- **Инструменты** — создание, правка, удаление, перемещение файлов; git-команды; grep/find; запуск команд в терминале
- **Параллельные tool calls** — при облачном API независимые операции выполняются одновременно (Promise.all)
- **Параллельные агенты** — каждый чат запускает и останавливает агента независимо
- **Суммаризация контекста** — при ≥85% лимита сжимает историю через Ollama
- **Адаптивные лимиты** — `maxToolMessageChars` и `maxHistoryMessages` масштабируются по окну контекста модели
- **Детектор опасных задач** — спрашивает подтверждение перед деструктивными операциями
- **Очередь сообщений** — до 50 запросов в очереди с watchdog и retry

### Модели
- **Ollama (локально)** — автовыбор модели по сложности задачи; эскалация при рефьюзале; скачивание из каталога прямо из настроек
- **DeepSeek** — нативный tool calling по OpenAI API
- **OpenAI** — совместимый адаптер; любой endpoint
- **Совместимость** — значки ✓/⚠ рядом с моделями; несовместимые по RAM отключены

### История чатов
- Папки с привязкой к проекту; поиск; drag-and-drop; pin; теги
- Режимы Chat / Code (переключатель в сайдбаре)
- Экспорт JSON / Markdown; импорт с merge по ID
- Автовосстановление после краша

### Память и обучение
- **Семантический поиск** — векторный поиск через `nomic-embed-text`; LRU-кеш 500 записей
- **Самообучение** — рефлексия после задач; план улучшений с инструментами `set/complete_self_improvement_plan`
- **Саморедактирование** — агент правит собственный код; в режиме самоулучшения — commit + push; вне режима — правки уходят в `git stash` и применяются при следующем запуске
- **Экспорт навыков и памяти** — GitHub Gist через API

### UI
- Контекст-бар с процентом заполнения, чипами лимитов и индикатором суммаризации
- Стриминг с `React.memo`; виртуализация длинных списков (`@tanstack/react-virtual`)
- Ленивая загрузка тяжёлых компонентов (`React.lazy`)
- Метрики в топбаре: время, токены, tok/s; GPU/CPU-режим
- Открытие файлов из чата — клик на путь открывает в Проводнике
- Терминал со встроенным автодополнением (история 200 команд)
- Темна тема (GitHub-dark палитра); горячие клавиши; звуковые уведомления; WCAG AA

### Безопасность
- `run_command` без `shell: true` — нет инъекций
- Шифрование API-ключей через `safeStorage`
- Санитизация путей (`assertInsideProject`) во всех инструментах
- Blocklist для запрещённых команд

### Инфраструктура
- Worker threads для grep/find, эмбеддингов, парсинга больших файлов — main process не блокируется
- ESLint + Prettier + lint-staged в husky
- Vitest (unit) + Playwright (e2e); GitHub Actions CI
- Branch protection на master; PR через `gh` CLI
